#背景

为了提升产品在复杂网络环境下的体验，性能，需要搭建一套模拟网络环境来提升效率，主要是通过Ubuntu建立热点小范围分享Ap，使用Facebook的开源项目ATC（Augmented Traffic Control）来进行流量控制。

##工具介绍
ATC （Augmented Traffic Control）是 Facebook 开源的增强网络流量控制工具ATC，能利用WiFi网络模拟各种移动网络，测试智能手机和APP在不同国家地区和应用环境下的性能表现。ATC能够模拟2G、2.5G（Edge）、3G和LTE4G网络环境，测试工程师可以快速在各种不同模拟网络环境中切换。 ATC能够模拟真实网络环境中的用户体验，Facebook开发ATC的目的是确保更多的人，尤其是发展中国家的用户也能够获得最好的APP体验。

##搭建步骤

###搭建环境：
系统：OS X 10.10.3
VM：virtualBox Linux: ubuntu 14.04

###安装ATC相关步骤：
检测环境：ifconfig查看是否识别无线网卡wlan0

1.搭建并配置热点wifi

执行：
sudo apt-get install dnsmasq
sudo apt-get install hostapd
sh wifi.sh #准备好的脚本文件
这里wifi就搭建好了，如果出现configure错误

则执行 sudo nmcli nm wifi off
sudo rfkill unblock wlan
sh wifi.sh #再次执行该文件即可 配置成功：

####2. 安装ATC：
sudo apt-get install python-pip git-core #安装pip和git
sudo pip install atc_thrift atcd django-atc-api django-atc-demo-ui django-atc-profile-storage #pip安装相关组件

#apt-get自带的django和bootstrap-themes都需要更新
sudo pip uninstall Django
sudo pip install –v Django==1.9.8
sudo pip uninstall Django-bootstrap-themes
sudo pip install Django-bootstrap-themes #下载atc git库，官方配置了一些选项，用来直接倒入数据库中
sudo git clone https://github.com.facebook/augmented-traffic-control.git
创建并修改文件生成atc UI库

####3. 创建UI目录

Django-admin startproject atcui 进入atcui目录，注意这里有两层目录，编辑setting.py在INSALLED _APPS里添加如下内容:

#Django ATC API ‘rest_framework’,
‘atc_api’,

Django ATC Demo UI

‘bootstrap_themes’,
‘django_static_jquery’,
‘atc_demo_ui’,

Django ATC Profile Storage

‘atc_profile_storage’,

编辑urls. py，倒入下面两个库，加入以下两条语句即可

from django.views.generic.base import RedirectView
from django.conf.urls import include 编辑urls.py，

在urlpatterns添加如下内容

urlpatterns = [
… # Django ATC API
url(r’^api/v1/‘, include(‘atc_api.urls’)),

Django ATC Demo UI url(r’^atc_demo_ui/‘,include(‘atc_demo_ui.urls’)),

Django ATC profile storage url(r’^api/v1/profiles/‘,

include(‘atc_profile_storage.urls’)),
url(r’^$’, RedirectView.as_view(url=’/atc_demo_ui/‘, permanent=False)),
]

####4. 生成数据库（atcui UI目录下）
python manage.py migrate 上面修改好，数据库这里就不会报错

####5. 启动atcd服务
sudo atcd –atcd-wan eth0 –atcd-lan wlan0 #使用网卡eth0作为公网连接，wlan0作为内网连接

####6. 运行ATC UI
python manage.py runserver 0.0.0.0:8000
导入facebook的相关配置选项
cd augmented-traffic-control #进入下载的facebook git库
utils/restore-profiles.sh localhost:8000

####7.结束
用手机连接电脑的ip：8000即可食用
示例：

##wifi.sh脚本文件内容
mkdir /etc/hostapd.conf
mkdir /etc/dnsmasq.conf

Define configure file path

HOSTAPD_CONF_PATH=”/etc/hostapd.conf”
DNSAMSQ_CONF_PATH=”/etc/dnsmasq.conf”

Turn down servces

service dnsmasq stop
service hostapd stop

Backup configure files

mv “$HOSTAPD_CONF_PATH” “${HOSTAPD_CONF_PATH}.old”
mv “$DNSAMSQ_CONF_PATH” “${DNSAMSQ_CONF_PATH}.old”

Read WiFi Setting

read -p “Please input your AP name : “ apName
read -p “Please input your AP password : “ apPwd

Modify hostapd configure file

echo “# Define interface” > “$HOSTAPD_CONF_PATH”

echo “interface=wlan0” >> “$HOSTAPD_CONF_PATH”

echo “# Select driver” >> “$HOSTAPD_CONF_PATH”

echo “driver=nl80211” >> “$HOSTAPD_CONF_PATH”

echo “# Set access point name” >> “$HOSTAPD_CONF_PATH”

echo “ssid=$apName” >> “$HOSTAPD_CONF_PATH”

echo “# Set access point harware mode to 802.11g” >> “$HOSTAPD_CONF_PATH”

echo “hw_mode=g” >> “$HOSTAPD_CONF_PATH”

echo “# Set WIFI channel (can be easily changed)” >> “$HOSTAPD_CONF_PATH”

echo “channel=6” >> “$HOSTAPD_CONF_PATH”

echo “# Enable WPA2 only (1 for WPA, 2 for WPA2, 3 for WPA + WPA2)” >> “$HOSTAPD_CONF_PATH”

echo “wpa=3” >> “$HOSTAPD_CONF_PATH”

echo “wpa_passphrase=$apPwd” >> “$HOSTAPD_CONF_PATH”

Modify dnsamsq configure file

echo “interface=wlan0” > “$DNSAMSQ_CONF_PATH”

echo “bind-interfaces” >> “$DNSAMSQ_CONF_PATH”

echo “dhcp-range=192.168.43.2,192.168.43.10” >> “$DNSAMSQ_CONF_PATH”

echo “dhcp-option=option:router,192.168.43.1” >> “$DNSAMSQ_CONF_PATH”

Open wlan0

ifconfig wlan0 up

Set wlan0 ip address

ifconfig wlan0 192.168.0.1

Configure NAT

Delete NAT rules

iptables -t nat -F

Add NAT rule

iptables -t nat -A POSTROUTING -o eth0 -j MASQUERADE

Enable ip forwoad

sysctl net.ipv4.ip_forward=1

start services

service dnsmasq start
service hostapd start

========= Open WIFI ========

echo -e “\n===== Open WiFi… =====\n”
hostapd “$HOSTAPD_CONF_PATH”
echo -e “\nWiFi Stop, Cleaning……\n”
